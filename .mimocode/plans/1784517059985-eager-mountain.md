# Plan: Recreate Auth Components

## Problem

Metro bundler fails with `Unable to resolve "@/components/auth/OrcaLogo"` because all 4 auth component files were deleted from `src/components/auth/`. The welcome screen (`src/app/index.tsx`) imports them but they don't exist.

## Components to Create

All files go in `src/components/auth/`. Each is a functional component with named export, PascalCase filename, NativeWind className styling — per AGENTS.md.

### 1. `OrcaLogo.tsx`
- **Purpose**: Brand logo displayed on welcome screen
- **Design**: Orca/whale shape in dark green (`#0A3622`), ~120x120pt, centered
- **Implementation**: Pure View-based composition (no SVG library available). Multiple nested Views with border-radius, rotation, and background colors to approximate the orca silhouette from `WELCOME TO DALALI.png`
- **Props**: None
- **Usage**: `<OrcaLogo />`

### 2. `GoogleIcon.tsx`
- **Purpose**: Google "G" icon for the "Continue with Google" button
- **Design**: Multicolor G (blue, red, yellow, green) on transparent background, ~20x20pt
- **Implementation**: View-based with colored segments arranged in a G shape
- **Props**: None
- **Usage**: `<GoogleIcon />`

### 3. `LockIcon.tsx`
- **Purpose**: Padlock icon for the "Continue with Password" button
- **Design**: White padlock outline, ~18x18pt (sits on dark green button background)
- **Implementation**: View-based with border/rounded views forming a lock shape
- **Props**: None
- **Usage**: `<LockIcon />`

### 4. `Checkbox.tsx`
- **Purpose**: Consent checkbox on welcome screen
- **Design**: Dark green circle (`#0A3622`) with white checkmark when checked, empty circle when unchecked. ~22x22pt
- **Implementation**: View with conditional checkmark (two small rotated Views forming a check)
- **Props**: `checked: boolean`
- **Usage**: `<Checkbox checked />`

## Design References

- TypeScript strict mode, no `any`
